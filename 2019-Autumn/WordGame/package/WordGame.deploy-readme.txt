
===========================
Предварительные требования :
--------------------------
Для развертывания этого веб-пакета на компьютере, на котором выполняется cmd-файл, должен быть установлен инструмент Web Deploy (msdeploy.exe). Сведения об установке инструмента Web Deploy см. по следующей ссылке:
https://go.microsoft.com/?linkid=9278654
Для этого пакетного файла требуется, чтобы файл пакета "WordGame.zip" и необязательно указываемый файл параметров "WordGame.SetParameters.xml" находились в той же папке или папке назначения по переменной среды.

===========================
Использование:
--------------------------
WordGame.deploy.cmd [/T|/Y] [/M:имя_компьютера] [/U:имя_пользователя] [/P:пароль] [/G:UseTempAgent] [Дополнительные параметры msdeploy.exe ...]


===========================
Обязательные флаги:
--------------------------
/T:
	Вызывает msdeploy.exe с флагом "-whatif", который имитирует развертывание. В этом случае пакет не развертывается. Вместо развертывания создается отчет по действиям, которые будут выполнены при действительном развертывании пакета.
/Y:
	Вызывает msdeploy.exe без флага "-whatif", при этом выполняется развертывание на текущем компьютере или на сервере назначения. Используйте флаг /Y после проверки данных, выданных при использовании флага /T.

Примечание: Не используйте одновременно оба флага /T и /Y в одной команде.

===========================
Необязательные флаги:
--------------------------
 По умолчанию этот сценарий выполняет развертывание на текущем компьютере, на котором он вызван, с учетными данными текущего пользователя без службы агента. Передавайте в расширенный сценарий только следующее значение.

/M:<имя конечного сервера или URL-адрес службы>
	Если этот флаг не указан, пакет устанавливается на том компьютере, на котором выполнена команда. URL-адрес службы может иметь следующий формат:
		https://<конечный_сервер>:8172/MSDeploy.axd 
	Для использования этого формата необходимо, чтобы на сервере назначения был установлен IIS 7, а также настроены служба веб-управления IIS 7 (WMSvc) и обработчик Web Deploy.
	URL-адрес службы также может иметь следующий формат:
		http://<конечный_сервер>/MSDeployAgentService 
	Для этого формата требуются административные права на сервере назначения, также на этом сервере должна быть установлена служба удаленного Web Deploy (MsDepSvc). Служба IIS 7 не обязательно должна быть установлена на конечном сервере.

/U:<имя_пользователя>
/P:<пароль>
/G:<True | False>
	Указывает, что пакет развертывается посредством создания временного прослушивателя на сервере назначения. Для этого не требуется выполнять отдельную установку на сервере назначения, однако необходимы права администратора на этом сервере. Значение этого флага по умолчанию равно False. 

/A:<Basic | NTLM>
\Задает используемый тип проверки подлинности. Возможные значения: NTLM и Basic. Если указан параметр поставщика wmsvc, по умолчанию используется тип проверки подлинности Basic. В противном случае используется тип проверки подлинности NTLM. 

/L 
	Указывает, что пакет развернут на локальном пользовательском экземпляре IISExpress.

[Дополнительные флаги msdeploy.exe]
	Команда msdeploy.exe поддерживает дополнительные флаги. Любой из этих дополнительных флагов можно включить в файл "$(ИмяПроекта).Deploy.cmd", тогда при выполнении эти флаги будут переданы в msdeploy.exe.
	Также дополнительные флаги можно указать, установив переменную среды "_MsDeployAdditionalFlags". Эти параметры используются данным пакетным файлом.
	Примечание. Любые значения флага, содержащие знак равенства (=), должны заключаться в двойные кавычки. Это показано в следующем примере, который пропускает развертывание баз данных, включенных в пакет:
		"-skip:objectName=dbFullSql" 

===========================
Зависимые от окружения свойства:
--------------------------

	Чтобы настроить зависящие от приложений параметры для каждой из сред развертывания (например, имя приложения IIS, физический путь и любые строки подключения), отредактируйте параметры в следующем файле: 
		 "WordGame.SetParameters.xml"
